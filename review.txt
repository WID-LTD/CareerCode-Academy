# CareerCode Academy - Email Verification System Implementation

You are working on the CareerCode Academy codebase. Implement a complete email verification system for all newly registered users.

## Objective

Users must verify their email address before they can access their dashboard, enroll in courses, create courses, or use any protected features.

---

## Requirements

### Registration Flow

When a user registers:

1. Validate all input fields.
2. Create the user account.
3. Hash the password using bcrypt.
4. Generate a secure verification token using Node.js crypto.
5. Store the verification token and expiration date in the database.
6. Set `emailVerified` to `false`.
7. Send a verification email to the user's email address.
8. Redirect the user to a "Check Your Email" page.

---

## Database Updates

Add the following fields to the User model:

```typescript
emailVerified: boolean;
verificationToken: string | null;
verificationTokenExpires: Date | null;
```

Default values:

```typescript
emailVerified = false
verificationToken = null
verificationTokenExpires = null
```

---

## Email Verification Endpoint

Create:

```typescript
GET /api/auth/verify-email
```

Workflow:

1. Receive verification token.
2. Find matching user.
3. Check token expiration.
4. If valid:

   * Set `emailVerified = true`
   * Remove verification token
   * Remove expiration date
5. Redirect user to verification success page.

Invalid or expired tokens should redirect to an error page with an option to resend verification email.

---

## Login Protection

Update login logic.

Before generating JWT:

```typescript
if (!user.emailVerified) {
  throw new Error("Please verify your email address.");
}
```

Unverified users must not:

* Access dashboard
* Enroll in courses
* Purchase courses
* Create courses
* Access protected routes

---

## Resend Verification Email

Create endpoint:

```typescript
POST /api/auth/resend-verification
```

Requirements:

1. Accept email address.
2. Find user.
3. Generate new verification token.
4. Update expiration date.
5. Send new verification email.
6. Return success response.

Add rate limiting to prevent abuse.

## Security Requirements

* Use crypto-generated tokens.
* Verification links expire after 24 hours.
* Tokens are single-use.
* Remove token after successful verification.
* Prevent login for unverified accounts.
* Add rate limiting on resend endpoint.
* Never expose sensitive user information.
